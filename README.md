# 🏢 수벨건물 입주자 관리 프로그램

수벨건물의 **14개 호실**을 통합 관리하는 설치형 웹앱(PWA)입니다.
별도 설치 없이 브라우저로 접속하고, **홈 화면에 앱으로 설치**할 수 있으며,
**오프라인에서도 동작**합니다. 모든 데이터는 사용 기기(브라우저)에 안전하게 저장됩니다.

> 빌드 과정이 없는 순수 정적 웹앱이라 GitHub Pages로 바로 배포됩니다.

---

## ✨ 주요 기능

### 상황판 (대시보드)
- 현 임대 호실 수 / 14, **가동률**
- **월 수입 총계** (월세 + 관리비 합산)
- **월 지출 총계** (상수도·정화조·공동전기·기타)
- **월 순수입** 및 보증금 합계

### 호실 관리 (14개 기본 호실)
- 기본 호실: 101 / 201~205 / 301~305 / 401~403
- 호실별 입력: 임차인 이름, 출생년, 직업, 최초 입주일, 만기일,
  보증금, 월세, 관리비, 선불/후불, 전세/월세, 세대 옵션
- **자동 계산**
  - 총액 = 월세 + 관리비
  - 거주개월 = 최초 입주일 기준 1달 후 도래일을 1개월로 산출
  - 누적 수령액(추정) = 거주개월 × 총액

### 만기 알림
- 만기 **2개월 전** 자동 알림
  - _"XXX호 만기 2개월 전입니다. 재계약 여부 확인 바랍니다."_
- 상단 🔔 배지 + 상단 배너로 즉시 확인
- **🔔 폰 알림(설치형 PWA)**: 설정 → "만기 폰 알림 켜기" 로 실제 안드로이드 시스템 알림 수신
  - 앱을 열 때 임박 호실이 있으면 즉시 알림 (하루 1회)
  - 설치된 PWA에서는 **백그라운드 주기 점검**(Periodic Background Sync)으로 앱을 닫아도 하루 1회 점검 — _Chrome/Android 한정, 발생 시각은 브라우저가 결정하는 best-effort_
  - ⚠️ 서버가 없는 온디바이스 앱이라, 서버에서 보내는 정시 푸시(앱을 오래 안 열어도 도착)는 지원하지 않습니다.

### 호실 상세 (행 클릭)
- **계약·임차인** 탭: 출생년, 직업, 선불/후불, 누적 총액 등
- **세대 옵션** 탭: 냉장고·세탁기·에어컨·전자레인지·인덕션·가스레인지·실링팬·소화기
  (항목 자유 추가 가능)
- **VOC 민원** 탭: 민원 내용 + 처리일자 + 비용 기록

### 추이 그래프 📈
- 최근 **12개월 월별 수입·지출·순수입** 라인 차트 (외부 라이브러리 없음)
- 월별 실적 직접 입력 가능, 미입력 월은 현재 값 자동 적용

### 추가 개선 사항 (제안 반영)
- 🔍 호실/임차인/직업 **검색** + 임대중·공실·만기임박 **필터**
- 🟢🟡🔴 호실 **상태 배지** (임대중 / 만기임박 / 만기경과 / 공실)
- 🏠 **공실 관리** 및 입주 정보 비우기
- 🏗️ **호실 추가·삭제** (설정 → 호실 추가·삭제)
- 💾 **JSON 백업/복원** (기기 변경 대비)
- 🖨️ 현황 **인쇄 / PDF 저장**
- 🌙 **다크 모드**
- 📲 **PWA 설치** + 오프라인 지원, 자동 저장

## 🚀 배포 (GitHub Pages + Actions)

이 저장소에는 자동 배포 워크플로(`.github/workflows/deploy.yml`)가 포함되어 있습니다.

> ⚠️ **최초 1회만 수동 설정이 필요합니다.** GitHub 보안 정책상 Actions 토큰이
> Pages 사이트를 자동 생성할 수 없어, 아래 한 번의 클릭이 필요합니다.

1. GitHub 저장소 → **Settings → Pages**
2. **Build and deployment → Source** 를 **GitHub Actions** 로 변경 (저장)
3. **Actions** 탭 → **Deploy to GitHub Pages** → 최근 실패한 실행에서 **Re-run jobs**
   (또는 아무 커밋이나 다시 푸시) → 자동 배포 완료

이후부터는 `main` 브랜치에 푸시할 때마다 자동 배포됩니다.

배포가 끝나면 다음 형태의 링크로 접속할 수 있습니다:

```
https://supervisor-515.github.io/monthly-rent/
```

> Actions 탭의 **Deploy to GitHub Pages** 실행 결과에서 최종 URL을 확인할 수 있습니다.

### 휴대폰에 앱으로 설치
- **Android(Chrome)**: 접속 후 메뉴 → "앱 설치" / "홈 화면에 추가"
- **iOS(Safari)**: 공유 → "홈 화면에 추가"

## 🖥️ 로컬 실행

빌드가 필요 없습니다. 정적 서버로 열기만 하면 됩니다.

```bash
python3 -m http.server 8080
# 브라우저에서 http://localhost:8080 접속
```

> 서비스워커(오프라인 기능)는 `http://localhost` 또는 `https://` 에서만 동작합니다.

## 🗂️ 구조

```
.
├── index.html               # 메인 화면
├── css/styles.css           # 스타일 (라이트/다크/인쇄)
├── js/calc.js               # 순수 계산·포매팅 함수 (테스트 대상)
├── js/notify-store.js       # 알림 공용 로직 + IndexedDB (페이지·SW 공유)
├── js/app.js                # 애플리케이션 로직 (바닐라 JS)
├── manifest.webmanifest     # PWA 매니페스트
├── sw.js                    # 서비스워커 (오프라인 캐시 + 백그라운드 만기 알림)
├── tests/                   # 단위 테스트 (의존성 없음)
├── icons/                   # 앱 아이콘 (SVG + PNG)
├── scripts/gen_icons.py     # 아이콘 생성 스크립트
└── .github/workflows/
    ├── ci.yml               # 단위 테스트 + 문법 검사
    └── deploy.yml           # Pages 자동 배포
```

## ✅ 테스트
순수 계산 함수(`js/calc.js`)는 외부 의존성 없이 Node 표준 라이브러리로 검증합니다.

```bash
node tests/calc.test.js
```

`main`·`claude/**` 브랜치 푸시와 PR에서 CI가 자동으로 테스트와 문법 검사를 실행합니다.

## 🔒 데이터 보관
모든 데이터는 브라우저 `localStorage`에 저장되며 서버로 전송되지 않습니다.
기기 분실·초기화에 대비해 **설정 ⚙️ → 데이터 백업**으로 정기적으로 내보내세요.
